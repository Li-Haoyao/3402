## Content Insights

One important thing I learned this week is the separation between HTML structure and CSS styling. HTML is mainly responsible for content and layout structure, while CSS controls appearance and presentation. This separation improves maintainability because the visual design can be changed without rewriting the page content.

I also learned that Bootstrap simplifies responsive web design significantly. Instead of manually writing large amounts of CSS, Bootstrap provides pre-built classes and layout systems such as containers, rows, and columns. Using these components made it much easier to create pages that adapt to different screen sizes.

Another useful concept I learned was website deployment using GitHub Pages. Before this week, I mainly viewed websites locally on my computer. Publishing a site online helped me understand how web hosting works and the importance of file structure. I learned that the homepage file must be named `index.html`, otherwise the deployment will not work correctly.

I also became more familiar with Markdown formatting while preparing the learning journal. Markdown is simpler than HTML but still allows clear formatting through headings, lists, code blocks, and links.

Example of a Bootstrap button used in my website:

```html
<a href="features.html" class="btn btn-primary btn-lg">
    Explore Features
</a>
```

This helped me understand how Bootstrap classes can quickly style elements without requiring custom CSS.

## Career/Employability/Learning Insights

This week made me realise that even entry-level web development requires more than just coding knowledge. Employers frequently expect developers to understand GitHub, version control, deployment, and responsive design in addition to writing HTML and CSS.

While researching Bootstrap and GitHub Pages, I noticed that many junior web developer job advertisements mention skills such as Git, responsive UI design, and familiarity with frontend frameworks. This showed me that learning deployment tools and development workflows is valuable for employability, even for relatively simple projects.

I also learned something about my own learning process. Watching long tutorials continuously was not very effective for me because I lost concentration and forgot information quickly. I found that I learned much faster when I alternated between short tutorials and immediately testing the concepts myself in VS Code. Writing code while learning helped me remember concepts more effectively and reduced the need to repeatedly rewatch tutorials.

Another thing I noticed is that building a real website, even a simple one, feels much more engaging than only reading documentation. Solving small practical issues, such as fixing broken CSS paths or deploying a website successfully, gave me more confidence in learning web technologies independently.
