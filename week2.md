## Content Insights

This week I learned that Joomla works very differently from a static HTML website. In Week 1, 
I needed to create separate HTML files for each page, such as `index.html`, `about.html`, and `contact.html`. 
In Joomla, the content is usually created as articles, and the navigation is controlled through menus.

One important concept I learned is the relationship between articles, categories, and menus. 
At first, I was confused because my homepage article appeared inside the `Uncategorised` category. 
I thought pages and menu items were the same thing, but I later realised that Joomla separates content from navigation. 
An article is the actual content, a category is used to organise articles, 
and a menu item is used to let users access that content from the website navigation.

I also learned that a CMS can make website management easier because I do not need to manually edit every HTML file. 
For example, if I change a menu item or template setting, it can affect the whole site. 
This is useful for larger websites because it saves time and keeps the layout more consistent.

Another useful thing I learned was template customisation. A default Joomla website does not look suitable for a startup business, 
so I needed to change the site name, menu structure, page content, and images. 
This helped me understand that a CMS still requires design decisions. Even though Joomla provides the system, 
the website still needs clear purpose, suitable text, and relevant images.

A simple structure for my Joomla startup website could be:

- Home page: Introduces the startup idea
- About page: Explains the purpose of the business
- Services or Features page: Shows what the startup provides
- Contact page: Provides basic contact information

This helped me understand that website planning is still important when using a CMS. The tool can help build the website, but it does not automatically create good content or a clear user experience.

## Career/Employability/Learning Insights

This week helped me understand why CMS skills are useful for employability. 
Many small businesses do not need a fully custom-coded website. 
They often need someone who can set up, customise, update, and manage a CMS website. 
Learning Joomla gave me a better idea of how real business websites can be created and maintained without writing every page from scratch.

I also realised that CMS work still requires technical understanding. At first, 
Joomla felt easier because I did not need to write much code, but I quickly found that it has its own learning curve. 
I needed to understand where articles, categories, menus, templates, and settings are located. 
This showed me that using a CMS is not just clicking buttons. It still needs logical thinking and problem-solving.

My learning process this week was not completely smooth. I became confused when trying to connect the homepage, articles, and menus. 
However, solving this problem helped me understand Joomla more clearly. Instead of only following steps, I started to understand the system structure behind the website.

I also learned that I need to spend more time practising outside class. The Week 2 task reminded me that if I fall behind in practical work, 
it becomes harder to understand the next task. For future weeks, I should try to finish the basic practical task earlier, then use extra time to improve the design and write a better learning journal.

Compared with Week 1, I feel that I am slowly moving from simple static web design to more realistic website management. 
This is useful because many workplace websites use CMS platforms, and being able to use both HTML/CSS and CMS tools can make me more flexible as a beginner web developer.
